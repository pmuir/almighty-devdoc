Design Posting Process

When posting your work, use the URL from InVision’s Share feature and not the URL from the browser.

NOTE: This URL will not change when you update the content of the prototype. It will change when you change the attributes on the prototype, such as allowing commenting.

Post design artifacts* and links

*(artifacts include materials related to the design, discovery, workflows, wireframes, prototypes, visuals…)

  1. Post InVision links in the UXD JIRA stories

  2. Post JIRA link in GIT story in almighty-ux

    • Once your story is pulled into an AppDev sprint, create an issue that includes a link to the JIRA story on https://github.com/almighty/almighty-ux/issues.

  3. Post artifact links in GIT devdocs

    1. Based on the type of artifact you want to link to, determine which ASCII doc** should be updated.

      • Wireframes: almighty-devdoc/ux/design/wireframes/wireframes.adoc

      • Workflows: almighty-devdoc/ux/design/workflows/workflows.adoc

      • Visuals: almighty-devdoc/ux/design/visuals/visuals.adoc

      • Discovery: almighty-devdoc/ux/discovery/discovery-overview.adoc

      • Strategy: almighty-devdoc/ux/strategy/strategy-overview.adoc

      • Usability: almighty-devdoc/ux/usability-testing/usability-overview.adoc

    2. Update the document to include a link to the artifact and a brief description of what it contains.

      ASCII docs can be edited using most code editors. We recommend Atom or VSCode.

Communicate that designs are ready to review

Once a design artifact (whether new or updated) has been posted to InVision, let the team know that it is ready for review.

  1. Post to HipChat

    • On the Almighty UI/UX channel using @all, the InVision link, and a brief message.

    • On the Almighty channel using @here, the InVision link, and a brief message.

  2. Post to email lists

    • Internal: Post here any sensitive materials to share with the internal group and/or when a design is in progress and you are working with development on specific solution issues: devtools-almighty@redhat.com

    • External: Post here when you have vetted the design through UXD and are ready for broader/public/community feedback: almighty-public@redhat.com

Communicate that designs are ready for dev

Once the design is complete and it has been reviewed and agreed upon by development and stakeholders, notify development that it’s ready.

  1. On the pertinent issue in GIT, add the "Ready for Dev" label.

  2. On that same issue, add a comment mentioning @joshuawilson and the developer on the feature.
